## Short description

> CodeWorld Observatory — a simulation-first control plane for agentic software engineering. It treats a repository as an observable causal world so consequences can be imagined, compared, and reviewed before any write.

## Longer description

> CodeWorld Observatory is a research scaffold for simulation-first agentic software engineering. Instead of generating code write-first, it models a software repository as a structured technical world — state, dependencies, histories, tests, fragile zones, possible futures, and prediction-versus-reality comparison — and keeps every output advisory and non-mutating.
>
> A separated Quantum Research Annex extends the thesis with quantum-inspired **resource accounting**: the Resource Consequence Ledger maps the discipline of explicit resource estimation onto software interventions (files affected, dependency zones, predicted failing tests, predicted vs. actual outcomes, uncertainty, human approval, rollback posture). Heavy quantum work stays in a local experiment layer; the web app renders exported JSON evidence only.

## What it proves

- Ability to design and ship a coherent, institution-grade Next.js application with a typed service layer and JSON API surface.
- A disciplined **simulation-before-write** governance thesis carried consistently across architecture, UI, and documentation.
- Restraint: a research annex that borrows a hard discipline (resource estimation) **without** overclaiming, fabricating results, or misrepresenting affiliations.
- Evidence-first engineering: exported, inspectable artifacts over assertions.

## What it does NOT claim

- **No quantum advantage** and no quantum speedup of any kind.
- **No Google affiliation, partnership, endorsement, or support.**
- No production quantum workloads; no live quantum compute in the web runtime.
- No autonomous execution; the system is observational and advisory only.
- No guarantees of correctness, optimality, or causal inference.
